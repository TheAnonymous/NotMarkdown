Adopt one document at a time

Fits the workflow you already have.

Start in the browser. Add the format to your repository only when it earns its place.

  1. 1
    Bring one Markdown file
    notmarkdown import README.md \
  --dialect commonmark --to nmdoc \
  --output README.nmdoc

    Explicit loss reports show what did not map—nothing disappears silently.

  2. 2
    Try it locally

    Open, edit, inspect, and save in the browser without creating an account or uploading a file.

  3. 3
    Keep Git readable
    notmarkdown git install --local .

    Package changes become reviewable canonical text in this repository only.

  4. 4
    Verify, then share
    notmarkdown verify handbook.nmdoc

    Source, assets, fallbacks, metadata, and integrity proofs travel as one inert file.

01Deterministic

Canonical source, member order, metadata, and compression profiles make identical inputs reproducible.

02Inspectable

CRC-32, SHA-256, byte sizes, codecs, and deferred verification state remain visible.

03Offline

No remote assets, cloud account, or server round-trip is part of document correctness.

04Inert

No scripts, macros, notebook cells, interactive forms, or arbitrary HTML.
